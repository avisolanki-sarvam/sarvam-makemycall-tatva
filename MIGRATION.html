What this migration does

Three tabs (Home / Campaigns / Settings) and the tab bar redesigned end-to-end against the Tatva token set, modelled after the CollectPay reference UX.

Visual rules locked in

  • White surfaces (#fafafa page, #ffffff cards), no shadows on cards
  • Indigo (#3333cc) only on brand tile, wallet pill, active tab
  • Green only on FAB and "Done" status pill
  • Red only on real "pending/failed" numeric tints
  • Settings rows: monochrome icon squares, label-over-value pattern
  • Phosphor regular weight (1.5px), constant in both tab states — colour alone signals active

Out of scope (untouched)

  • Auth flow (login, OTP, profile-setup)
  • Contacts, agents, campaign wizard, call detail screens
  • Native (android/, ios/) build configs
  • Backend API contracts (no shape changes)
  • Auth store, services/api.ts, contactImport.ts

    Done. tsc --noEmit --skipLibCheck green.

  4. 4

    Replace the node_modules symlink with a real install

    The current node_modules is a symlink to the original folder so the typecheck could run. Resolve dependencies for real before native builds.

    cd sarvam-makemycall-tatva
rm node_modules
npm install
  5. 5

    Clean the Android build cache (do NOT prebuild)

    Don't run expo prebuild --clean on this project. The android/ folder is intentionally committed with manual Firebase wiring (the google-services Gradle plugin in app/build.gradle, MainActivity.kt imports, etc.). prebuild --clean wipes and regenerates that folder from app.json, which loses the Firebase setup and crashes login with RNFBAppModule not found. Just clear Gradle's cache instead — Expo's autolinking picks up the new react-native-svg native module on the next install. 

    cd android && ./gradlew clean && cd ..

    If someone has already prebuild-wiped the folder, recover with: git checkout HEAD -- android/, then make sure android/app/google-services.json is present (it's gitignored — pull from _secrets/) before rebuilding.

  6. 6

    Run on Android device, do visual QA

    Compare side-by-side with MOCKUP.html. Walk the auth flow + agent creation + campaigns + settings tabs.

    npx expo run:android
  7. 7

    Sweep the un-migrated screens

    Auth, profile-setup, agents, campaigns wizard, contacts — all picking up the new palette via the COLORS shim. List anything that looks off; we redesign in a follow-up pass.

  8. 8

    Delete the ambient .d.ts

    Once npm install has succeeded and Phosphor's own types are present, remove the stub:

    rm src/types/phosphor-react-native.d.ts
  9. 9

    Promote the fork → main folder

    Either swap the folder names (recommended), or cherry-pick the diff into the original sarvam-makemycall.

    cd ..
mv sarvam-makemycall sarvam-makemycall-pre-tatva
mv sarvam-makemycall-tatva sarvam-makemycall

Decisions baked in

!
Brand colour change. The previous codebase enforced "no indigo / blue / purple anywhere" (cream + ink editorial). Tatva ships monochrome chrome with indigo as the single accent. The fork follows Tatva: indigo on brand mark, wallet pill, active tab. Update CLAUDE.md when promoting the fork so the rule reads "Tatva tokens only" rather than "no indigo".
✓
Icon pack: Phosphor v3. Picked over Lucide / HugeIcons because (1) regular-weight stroke matches Tatva's editorial restraint, (2) the React Native binding is mature, (3) Phosphor's web font is trivial to mirror in the mockup so design and code track each other 1:1. Note: v3 deprecated the un-suffixed names (House, Phone, …) in favour of HouseIcon, PhoneIcon, …. The fork uses the suffixed form throughout — no compiler strikethroughs.
⚠
Tab icons: constant weight. Active state communicated by colour alone. We initially toggled regular ↔ fill on focus; the negative-space change made the icon look like it shrank on press. Don't reintroduce.
